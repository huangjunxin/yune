# WEB-03 Phrase-Composition Regression Fix

Date: 2026-06-28 local

Verdict: success for the WEB-03 follow-up correctness fix. The browser memory
remeasure at `d4d84203` was valid for the launch path memory claim, but the
regenerated byte-backed Jyutping assets exposed a compact-storage behavior
regression: multi-syllable phrase composition and visible prefix lookup rows
were not both preserved.

## Root Cause

The compact byte-backed translator used the `Rime::Prism/4.0` payload for the
whole lookup code, but two fallback paths still probed the storage with raw
tone-stripped strings:

- sentence composition checked substrings such as `ngo`, `gok`, and `dak`
  directly against compact storage, where only canonical toneful codes such as
  `ngo5`, `gok3`, and `dak1` exist;
- prefix fallback checked prefixes such as `zou` and `zouhap` directly,
  allowing the regenerated compact path to miss or mis-rank visible dictionary
  rows.

The fix resolves non-correction prism aliases for sentence substrings and
prefix fallback fetches while preserving TypeDuck ordering:

- longest consumed prefix wins;
- within the same consumed prefix length, dictionary weight ranks rows;
- equal-weight ties keep original emission order, preserving legacy rows such
  as `ngo5` before `ngo4`.
