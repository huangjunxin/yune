<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser JS host -> TypeDuck C/WASM adapter | Untrusted JS caller passes pointers, keycodes, paths, and lifecycle calls into Rust ABI. |
| Browser virtual filesystem -> RIME runtime | MEMFS/IDBFS paths and preloaded files determine schema/config/dictionary availability. |
| Native fallback tests -> Phase completion | Local tests prove adapter contract when browser tooling is unavailable. |
| Documentation -> later phase implementation | Docs determine how Phase 8-10 callers interpret lifecycle, persistence, and fallback limits. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-03-01 | Tampering | Browser filesystem paths | high | mitigate | Tests and docs require shared data, user data, and `user_data_dir/build` to exist before init; docs keep runtime resource identifiers logical and host-created. |
| T-07-03-02 | Denial of Service | Missing preloaded assets | high | mitigate | Add negative native coverage for missing assets returning null and document that schema/dictionary assets must be preloaded before init. |
| T-07-03-03 | Tampering | Process-global lifecycle | high | mitigate | Preserve serialized tests and document one active process-global service; `yune_typeduck_cleanup` finalizes global service state. |
| T-07-03-04 | Repudiation | Fallback validation | high | mitigate | Docs and final gates require `./scripts/typeduck-wasm-build.sh` to distinguish verified browser build from blocker-plus-native-fallback output. |
| T-07-03-05 | Information Disclosure | Browser persistence | medium | mitigate | Docs state persistence sync remains JS host responsibility until Phase 9; Phase 7 adds no remote calls, secrets, or hidden data sync. |
| T-07-03-06 | Misleading Validation | Current scope docs | high | mitigate | Remove the old claim that Emscripten export-list/linker configuration is missing after Phase 7 while preserving explicit deferrals for TypeScript, IDBFS orchestration, E2E, multi-instance isolation, and AI-native work. |
</threat_model>
