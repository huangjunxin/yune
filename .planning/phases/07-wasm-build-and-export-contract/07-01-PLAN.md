<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer shell -> build tooling | Local commands execute toolchain binaries from PATH. |
| Export list -> linker flags | Plain text symbol names become Emscripten export flags. |
| Documentation -> executor/developer behavior | Incorrect docs can cause false validation or unsafe host assumptions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-01-01 | Tampering | scripts/typeduck-exports.txt | high | mitigate | Block if the export list includes non-adapter symbols or omits any D-04 symbol; verification checks exact count and required names. |
| T-07-01-02 | Spoofing | Toolchain command path docs | medium | mitigate | Docs name the intended script path and require actionable blockers for missing `wasm32-unknown-emscripten`, `emcc`, and `emar` rather than trusting ambiguous local state. |
| T-07-01-03 | Denial of Service | Emscripten optimized artifact | high | mitigate | Docs require `EXPORTED_FUNCTIONS` with all underscore-prefixed adapter names so optimization cannot silently remove JS-callable functions. |
| T-07-01-04 | Information Disclosure | Browser filesystem docs | medium | mitigate | Keep docs scoped to local MEMFS/IDBFS paths and do not add network, secrets, or remote service assumptions. |
| T-07-01-05 | Repudiation | Build/export validation docs | high | mitigate | Block on misleading validation: missing tools must be reported as blocker output and must not be documented as browser build success. |
</threat_model>
