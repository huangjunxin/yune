<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer shell -> `scripts/typeduck-wasm-build.sh` | Local script executes Cargo and toolchain commands. |
| PATH -> toolchain binaries | `cargo`, `rustup`, `nm`, `emcc`, `emar`, `wasm-nm`, and `wasm-objdump` are resolved from PATH. |
| Export list -> command-line linker flags | Reviewable text symbols are transformed into Emscripten `-s` flags. |
| Generated artifacts -> validation result | Artifact inspection determines whether browser export contract is valid. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07-02-01 | Spoofing | PATH toolchain detection | high | mitigate | Block on ambiguous missing tools by checking explicit commands and printing named blockers for missing target, `emcc`, or `emar`; do not silently pass browser validation. |
| T-07-02-02 | Tampering | Export flag generation | high | mitigate | Generate `EXPORTED_FUNCTIONS` only from `scripts/typeduck-exports.txt`; reject empty list; verify native and WASM artifacts against the same list. |
| T-07-02-03 | Repudiation | Script output | high | mitigate | Script must print either `TypeDuck WASM build verified:` or `TypeDuck WASM build blocked:` and run native fallback tests when blocked so logs distinguish success from fallback. |
| T-07-02-04 | Denial of Service | Command execution/path handling | medium | mitigate | Resolve repository root relative to script path, quote paths, avoid `/Users/trenton/` literals, and exit nonzero on failed native export or fallback tests. |
| T-07-02-05 | Information Disclosure | Generated artifact scan | low | accept | Artifact names and symbols are local build metadata with no secrets; script must not read `.env` or credential files. |
| T-07-02-06 | Elevation of Privilege | Toolchain installation | medium | mitigate | Script detects missing target/tools and prints install instructions; it must not install emsdk, run curl installers, or mutate global toolchains automatically. |
</threat_model>
