<interfaces>
Current risky path construction patterns to replace or guard:
```rust
fn normalize_config_resource_id(config_id: &str) -> String {
    config_id.strip_suffix(".yaml").unwrap_or(config_id).to_owned()
}

fn config_file_path(root: &str, resource_id: &str) -> PathBuf {
    Path::new(root).join(format!("{resource_id}.yaml"))
}

fn user_dict_path(dict_name: &str) -> PathBuf {
    runtime_user_data_dir().join(format!("{dict_name}.userdb"))
}

fn user_dict_snapshot_path(dict_name: &str) -> PathBuf {
    runtime_user_data_sync_dir().join(format!("{dict_name}.userdb.txt"))
}
```

Create a focused module with contracts equivalent to:
```rust
pub(crate) fn validate_config_resource_id(id: &str) -> Option<String>;
pub(crate) fn validate_data_resource_id(id: &str) -> Option<String>;
pub(crate) fn validate_user_dict_name(id: &str) -> Option<String>;
```
Return the normalized logical ID on success. Return `None` on empty strings, `.`/`..`, `../`, `..\\`, absolute Unix paths, Windows drive prefixes like `C:foo` or `C:\\foo`, backslashes, forward slashes, NUL, `~`, and extension-smuggling/filesystem path syntax not explicitly accepted. Config IDs may accept and strip a terminal `.yaml` only after rejecting path syntax. Schema config IDs may accept and strip a terminal `.schema`/`.schema.yaml` only if the remaining logical ID is safe. User dict names must be logical names only; do not accept `.userdb`, `.userdb.txt`, slashes, or snapshot paths as names.
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| C ABI caller -> config/schema APIs | Caller-provided logical IDs enter runtime config path construction. |
| Schema YAML -> runtime data/config loader | Schema-controlled dictionary/import/custom-settings IDs become file lookups. |
| C ABI caller -> userdb APIs | Caller-provided dictionary names and snapshot-derived names become user data paths. |
| Runtime roots -> filesystem | Validated logical IDs are joined with shared/user/staging/sync roots. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Tampering | config/schema resource IDs | mitigate | Validate logical IDs before `config_file_path` and reject traversal, separators, absolute paths, drive prefixes, NUL, and tilde paths. |
| T-02-03-02 | Information Disclosure | schema dictionary/import/vocabulary IDs | mitigate | Reject schema YAML values that would read outside runtime roots before `selected_runtime_data_path`. |
| T-02-03-03 | Tampering | userdb backup/export/import/restore destination names | mitigate | Validate user dictionary logical names before joining with user data or user sync directories; restore validates derived snapshot dict name before destination write. |
| T-02-03-04 | Elevation of Privilege | deployment file names | mitigate | Reject unsafe deploy config/schema logical filenames before copying from shared data to staging directories. |
| T-02-03-05 | Denial of Service | invalid resource inputs | mitigate | Fail closed using existing ABI conventions (`FALSE`, `-1`, `None`, `Value::Null`) without panics. |
| T-02-03-06 | Repudiation | validation coverage | mitigate | Add focused tests enumerating allowed and rejected IDs for config, schema/dictionary data, deployment, and userdb boundaries. |
</threat_model>
