<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Native/frontend caller -> lifecycle ABI | Untrusted caller order and repetition enters setup, initialize, finalize, and session APIs. |
| Runtime global state -> sessions/notifications | Process-global registries must not retain stale or cross-test state unexpectedly. |
| ABI callback -> frontend function pointer | Yune invokes caller-provided notification handler with temporary C strings. |
| Deployment/schema lifecycle -> notifications | Runtime emits externally observable callback order during deploy and schema changes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Denial of Service | repeated initialize/finalize | mitigate | Add deterministic loop-count regression tests and make lifecycle calls leave the service reusable without panics or poisoned locks. |
| T-02-02-02 | Tampering | session registry | mitigate | Assert destroyed and cleanup-all sessions are removed and cannot be used as stale handles. |
| T-02-02-03 | Repudiation | notification ordering | mitigate | Assert exact callback sequences for option/property/schema/deploy events so ordering regressions are observable. |
| T-02-02-04 | Elevation of Privilege | notification callback lifetime | mitigate | Keep callbacks outside internal locks, keep temporary C strings valid only during callback, and avoid storing frontend callback string pointers beyond invocation. |
| T-02-02-05 | Information Disclosure | process-global state pollution | mitigate | Reset or overwrite runtime/session/notification state through public lifecycle paths in tests; do not expose raw pointer values in diagnostics. |
| T-02-02-06 | Denial of Service | panic crossing FFI boundary | mitigate | Regression tests treat panic from public ABI lifecycle calls as failure; fixes return false/null or deterministic no-op where the ABI contract expects it. |
</threat_model>
