Dynamic loader symbol type to use, based on the exported symbol above:
```rust
type RimeGetApi = unsafe extern "C" fn() -> *mut yune_rime_api::RimeApi;
```

Use `libloading` with a null-terminated symbol name:
```rust
let get_api: libloading::Symbol<RimeGetApi> = unsafe { library.get(b"rime_get_api\0")? };
```

Keep the `Library` value alive for the full lifetime of any loaded symbol or table use. Do not transmute symbol lifetimes for this harness.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Native loader -> dynamic library | Test process loads a platform dynamic artifact and resolves exported C symbols. |
| Dynamic symbol -> unsafe Rust ABI table | `rime_get_api` returns raw pointer to process-global function table. |
| Frontend-like caller -> RimeApi callbacks | Harness invokes unsafe C ABI function pointers with caller-owned structs and C strings. |
| Test temp paths -> runtime setup | Harness supplies shared/user/staging paths to runtime globals. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Spoofing | dynamic artifact discovery | mitigate | Locate the Cargo-built `yune-rime-api` artifact under the active target directory and fail closed if the expected platform filename is absent; do not load arbitrary user-provided paths. |
| T-02-01-02 | Tampering | `rime_get_api` symbol lookup | mitigate | Resolve only the null-terminated `rime_get_api` symbol and verify the returned table and required function pointers are non-null before use. |
| T-02-01-03 | Denial of Service | panic crossing FFI boundary | mitigate | Harness assertions wrap call sites outside the dynamic library boundary and require runtime calls to return documented false/null results instead of relying on panics. Any panic during the dynamic test is an in-scope Phase 2 failure. |
| T-02-01-04 | Information Disclosure | failure diagnostics | mitigate | Error messages may include artifact names and target-dir-relative paths but must not dump arbitrary environment variables, temp directory contents, or pointer values. |
| T-02-01-05 | Elevation of Privilege | unsafe function-table invocation | mitigate | Use ABI structs from the crate, initialize `data_size` correctly, keep C strings alive across calls, and pair allocated outputs with matching free functions. |
| T-02-01-06 | Repudiation | validation findings | mitigate | Record each observed loader failure with command, observed behavior, expected behavior when known, scope decision, and target regression file. |
</threat_model>
