<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local schema YAML -> schema installation | Deployed schema YAML controls processor, segmentor, translator, and filter chains; treat it as untrusted local configuration. |
| frontend ABI -> key-processing session | `RimeProcessKey` receives frontend-driven key events and mutates session composition, candidates, and commits. |
| schema resource ID -> runtime filesystem lookup | Dictionary/config identifiers must remain logical IDs and must not become arbitrary filesystem paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01-01 | Tampering | `schema_install.rs` | mitigate | Preserve `validate_data_resource_id` and selected runtime data path helpers when adding schema-chain behavior; do not concatenate unvalidated schema strings into filesystem paths. |
| T-03-01-02 | Denial of Service | `speller.rs` previous-match recursion | mitigate | Bound recursive/iterative previous-match splitting by input length and consume input progress on each iteration; add tests using finite key sequences only. |
| T-03-01-03 | Elevation of Privilege | `lib.rs` dispatch | mitigate | Keep owned behavior in processor modules and avoid new ABI exports; if dispatch glue changes, it must be covered by ABI-facing tests before modification per D-03. |
| T-03-01-04 | Information Disclosure | ABI context/commit strings | accept | Tests inspect local composition/commit state only; no secrets or external services are introduced. |
</threat_model>
